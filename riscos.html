Norcroft 64

(date: 2025-09-23)

I’ve taken a couple of embedded projects from ARM32 to AArch64. The biggest time sinks weren’t the compiler, but hidden 32-bit assumptions: pointer math in ints, varargs, packed structs, and unions. Rewriting inline asm and aligning to the AAPCS64 calling convention was unavoidable. What helped most was harsh warnings, UBSan/ASan in tests, and swapping types to size_t/uintptr_t early. For legacy apps we used a small thunk layer and kept a 32-bit build running until parity.

Compo vs Gimp / ImageMagik?

(date: 2025-09-18)

Since you’ve asked three times for a Linux solution:

magick in.jpg -deskew 40% -trim +repage -colorspace HSL -channel s -evaluate set 0 +channel -colorspace Gray -normalize -level 2%,95% -sigmoidal-contrast 6x30% out.jpg

Works well on the first brown book scan I found: https://thumbs.dreamstime.com/z/text-old-book-3827559.…

Batch it with:

#!/usr/bin/env bash
set -euo pipefail
#
in_dir="pages"          # change if needed
out_dir="cleaned"
mkdir -p "$out_dir"
#
shopt -s nullglob
# Match common scan formats (case-insensitive)
for f in "$in_dir"/*.[Jj][Pp][Gg] "$in_dir"/*.[Jj][Pp][Ee][Gg] "$in_dir"/*.[Pp][Nn][Gg] "$in_dir"/*.[Tt][Ii][Ff] "$in_dir"/*.[Tt][Ii][Ff][Ff]; do
  base=$(basename "$f")
  base_noext="${base%.*}"
#
  # Keep anti-aliased greys, make paper white
  magick "$f" \
    -deskew 40% \
    -trim +repage \
    -colorspace HSL -channel s -evaluate set 0 +channel -colorspace Gray \
    -normalize \
    -level 2%,95% \
    -sigmoidal-contrast 6x30% \
    "$out_dir/${base_noext}.jpg"
done
